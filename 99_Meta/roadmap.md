# Roadmap for Continuous Expansion

The vault grows by accumulation. New material integrates as source notes and
release notes. The structure holds; knowledge does not fragment.

## Milestones

| Milestone | Scope | State |
|---|---|---|
| M1 | Foundation: folders, schema, templates, 23 MOCs, 22 seed notes, pipeline design. | in progress |
| M2 | Anthropic corpus depth: Claude SDK, Claude Code, MCP guides and code examples. | planned |
| M3 | Full Boris Cherny extraction: talks, posts, interviews, reusable principles. | planned |
| M4 | Release chronology backfill: every major Anthropic release, oldest to newest. | planned |
| M5 | Architecture deep-dives and multi-agent case studies. | planned |
| M6 | Evaluation and observability tooling guides. | planned |

## Integration rule

A new Anthropic release enters as a release note from
`90_Templates/template - release-note.md`, links its engineering implications to
existing concepts, and updates `[[MOC - Release Notes]]`. No restructure.

## Growth cadence

- Daily: triage inbox, run one or two channels, enrich stubs.
- Weekly: run all active channels, refresh MOCs, check orphans and links.
- On release: ingest the release note.

## Health signals

The vault dashboard (see `[[MOC - Home]]`) tracks stub count, orphan count, and
notes added in the last seven days.
